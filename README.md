# 学生考勤系统

## 项目简介

这是一个功能强大的学生考勤管理系统，专为教师设计，帮助您轻松记录和管理学生的出勤情况。系统支持多种便捷功能，如批量添加学生、批量设置出勤状态、单独管理每个学生的课时等。

## 技术栈

- **前端框架**: React 18.2.0
- **开发语言**: TypeScript 5.2.2
- **构建工具**: Vite 5.0.8
- **路由管理**: React Router 7.13.0
- **状态管理**: React Hooks (useState, useEffect)
- **数据存储**: 浏览器 LocalStorage
- **样式**: CSS3 (响应式设计, 动画效果)

## 功能特性

### 课程管理
- 创建新课程
- 查看课程列表
- 删除课程

### 学生管理
- 单个添加学生
- 批量添加学生（支持为每个学生单独指定课时）
- 编辑学生总课时
- 批量删除学生

### 考勤管理
- 为每个学生记录出勤状态（出勤、缺勤、迟到）
- 批量设置出勤状态
- 添加考勤日期
- 批量删除考勤日期

### 课时管理
- 为每个学生单独设置总课时
- 自动计算已上课时和剩余课时
- 基于出勤记录自动更新课时信息

### 数据持久化
- 使用浏览器 LocalStorage 存储数据
- 刷新页面或重新打开浏览器后数据不丢失

## 如何启动项目

### 前提条件
- 已安装 Node.js (推荐 v16+)
- 已安装 npm 或 yarn

### 步骤

1. **克隆项目**（如果尚未克隆）
   ```bash
   git clone <项目地址>
   cd student-attendance-system
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动开发服务器**
   ```bash
   npm run dev
   ```

4. **访问项目**
   打开浏览器，访问终端中显示的地址（通常是 `http://localhost:5173`）

### 其他可用命令

- **构建生产版本**
  ```bash
  npm run build
  ```

- **预览生产构建**
  ```bash
  npm run preview
  ```

- **代码检查**
  ```bash
  npm run lint
  ```

## 如何部署到互联网

以下是几种常见的部署方式，按难度从低到高排序：

### 方法 1: Vercel (推荐)

Vercel 是一个免费的静态网站托管平台，非常适合部署 React 应用。

1. **注册 Vercel 账号**
   访问 [Vercel](https://vercel.com/) 并使用 GitHub 账号注册

2. **连接 GitHub 仓库**
   - 登录 Vercel 后，点击 "New Project"
   - 选择 "Import Git Repository"
   - 连接到您的 GitHub 账号
   - 选择您的学生考勤系统仓库

3. **配置部署**
   - 项目名称: 可自定义
   - Framework Preset: 选择 "React"
   - Build Command: 保持默认 (`npm run build`)
   - Output Directory: 保持默认 (`dist`)
   - 点击 "Deploy"

4. **访问部署后的网站**
   部署完成后，Vercel 会提供一个 URL，您可以通过该 URL 访问您的应用

### 方法 2: GitHub Pages

1. **构建项目**
   ```bash
   npm run build
   ```

2. **安装 gh-pages**
   ```bash
   npm install --save-dev gh-pages
   ```

3. **修改 package.json**
   在 `scripts` 部分添加：
   ```json
   "predeploy": "npm run build",
   "deploy": "gh-pages -d dist"
   ```

4. **部署**
   ```bash
   npm run deploy
   ```

5. **访问网站**
   部署完成后，您的应用将可通过 `https://<your-github-username>.github.io/<repository-name>` 访问

### 方法 3: Netlify

1. **注册 Netlify 账号**
   访问 [Netlify](https://www.netlify.com/) 并注册

2. **部署项目**
   - 登录后，点击 "Add new site"
   - 选择 "Import an existing project"
   - 连接到您的 GitHub 账号
   - 选择您的仓库

3. **配置部署**
   - Build command: `npm run build`
   - Publish directory: `dist`
   - 点击 "Deploy site"

4. **访问网站**
   部署完成后，Netlify 会提供一个 URL

## 使用说明

### 添加课程
1. 在首页点击 "添加课程" 按钮
2. 填写课程名称、描述和时间段
3. 点击 "添加课程" 完成创建

### 添加学生
1. 选择一个课程
2. 在 "批量添加学生" 文本框中输入学生信息
3. 格式：每行一个学生，格式为 "姓名:课时"（例如：张三:15）
4. 点击 "批量添加" 按钮

### 记录考勤
1. 选择一个课程
2. 在日期选择器中选择日期
3. 点击 "添加考勤日期" 按钮
4. 为每个学生选择出勤状态（出勤、缺勤、迟到）
5. 或使用 "全部出勤"、"全部缺勤"、"全部迟到" 按钮批量设置

### 管理课时
1. 选择一个课程
2. 在学生姓名下方点击 "编辑" 按钮修改总课时
3. 系统会自动计算已上课时和剩余课时

### 删除操作
1. **删除学生**: 勾选学生前的复选框，点击 "删除选中学生" 按钮
2. **删除日期**: 勾选日期前的复选框，点击 "删除选中日期" 按钮
3. **删除课程**: 在课程列表中点击 "删除" 按钮

## 注意事项

1. **数据存储**: 系统使用浏览器 LocalStorage 存储数据，数据仅存储在当前浏览器中，不同浏览器或设备间不共享数据

2. **数据安全**: 由于使用 LocalStorage，不建议存储敏感信息

3. **数据备份**: 定期导出数据（可通过浏览器开发者工具查看 LocalStorage 内容）

4. **浏览器兼容性**: 推荐使用 Chrome、Firefox、Edge 等现代浏览器

5. **性能优化**: 对于大量学生和考勤记录，可能会出现性能下降，建议按班级或学期创建多个课程

6. **部署提示**: 部署到互联网后，所有访问者将共享同一套数据（存储在服务器端），请注意数据隐私

## 故障排除

### 问题: 页面刷新后数据丢失
**解决方案**: 检查浏览器是否禁用了 LocalStorage，或是否使用了隐私浏览模式

### 问题: 批量操作不生效
**解决方案**: 确保您已正确选择了要操作的学生或日期

### 问题: 课时计算不正确
**解决方案**: 检查学生的出勤记录，确保状态设置正确（只有 "出勤" 状态会计算课时）

### 问题: 部署后网站无法访问
**解决方案**: 检查部署配置是否正确，确保 `npm run build` 命令能成功执行
